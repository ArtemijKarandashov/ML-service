# Таблицы эндпоинтов
## 1. storage-service
* В продакшн версии доступ из вне к нему заблокирован
* Базовый префикс: `/api/v1/storage`

| Метод | Эндпоинт | Описание | Тело запроса | Успешный ответ | Ошибки | Взаимодействие |
|----------|-------------------------|--------------------------------------------|------------------------------------------------------------|----------------------------------------------------------------------------------------------|------------------------------------------------------------------------|----------------------------|
| `GET`    | `/`                     | Корневой эндпоинт сервиса                  | -                                                          | `200 OK`<br>`{"message": "Storage service. Probably would show some data here later."}`      | -                                                                      | - |
| `GET`    | `/get/{uid}`            | Скачать .pkl файл по его `uid`           | -                                                          | `200 OK`<br>`FileResponse` (сам файл)                                                        | `404` - запись не найдена<br>`409` - ошибка доступа к файлу<br>`500` - ОС ошибка | `client-service` |
| `POST`   | `/store`                | Загрузить новый .pkl файл, сохранить на диск, вернуть `uid` | `multipart/form-data`<br>`pkl_file: UploadFile`           | `200 OK`<br>`{"message": "File saved", "uid": "<uuid>"}`                                    | `404` - не удалось создать файл<br>`409` - дубликат `uid` или права<br>`500` - ОС ошибка | `client-service` |
| `DELETE` | `/delete/{uid}`         | Удалить .pkl файл и запись о нём         | -                                                          | `200 OK`<br>`{"message": "Deleted .pkl file", "uid": "<uid>"}`                              | `404` - запись не найдена<br>`500` - ошибка удаления                  | `client-service` |

## 2. client-service
* Сервис для общения с пользователем. Эти эндпоинты должны использоваться для фронтэнда
* Базовый префикс: `/api/v1`
* Client-service также должен отправлять пользователю фронтэнд, поэтому относительный вызов эндпоинтов в js (Без прописывания http://...) возможен.

### 2.1 Роутер `trainer`

* Префикс: `/trainer`
* Пример обращения: `/api/v1/trainer/upload-model`

| Метод | Эндпоинт | Описание | Тело запроса | Успешный ответ | Ошибки | Взаимодействие |
|----------|--------------------------|--------------------------------------------------------------------------|-----------------------------------------------------------|--------------------------------------------------------------------------------------|------------------------------------------------------------------------|------------------------------------------------------------------------------------------------|
| `POST`   | `/upload-model`          | Принимает .pkl файл, сохраняет его через storage‑service, возвращает `uid`. | `multipart/form-data`<br>`file: UploadFile` (`.pkl`)      | `{"message": "File saved", "uid": "<uid>"}`                                         | `409` - не удалось сохранить у storage<br>`415` - ошибка кодировки       | Вызывает `POST /api/v1/storage/store` у storage‑service, передаёт файл, получает `uid` и возвращает клиенту. |
| `GET`    | `/get-model/{uid}`       | Получает .pkl файл из storage‑service по `uid` и отдаёт клиенту.        | -                                                         | `200`<br>файл (`application/octet-stream`)                                          | `409` - storage не отдал файл<br>`415` - ошибка кодировки                 | Вызывает `GET /api/v1/storage/get/{uid}` у storage‑service, проксирует ответ клиенту.        |
| `GET`    | `/info-model/all`         | Отдает список всех доступных на данный момент моделей. (Нужно для таблички на фронтэнде)                   | -                               | `{ "models": {"{uid}": {"name": name, "status": status}, ...}}` | - | Пока не придумал как это правильне реализовать
| `DELETE` | `/delete-model/{uid}`    | Удаляет модель через storage‑service.                                     | -                                                         | `{"message": "Deleted .pkl file", "uid": "<uid>"}`                                  | `409` - storage не удалил<br>`415` - ошибка кодировки                     | Вызывает `DELETE /api/v1/storage/delete/{uid}` у storage‑service, возвращает ответ клиенту. |

### 2.2 Роутер `predictor` (планируемые эндпоинты, пока заглушки)
* Префикс: `/predictor`
* Пример обращения: `/api/v1/predictor/predict`

| Метод | Эндпоинт | Описание | Тело запроса | Успешный ответ | Ошибки | Взаимодействие |
|----------|--------------------------|--------------------------------------------------------------------------|------------------------------------------------------------------------------|-----------------------------------------------------------------|------------------------------------------------------------------------|----------------------------------------------------------------------------------|
| `GET`    | `/`                      | Информационный эндпоинт                                                  | -                                                                            | `{"message": "Predictor API functions are available on this route"}` | -                                                                      | -                                                                               |
| `POST`   | `/predict`               | Принимает `uid` модели и факторы клиента > возвращает статус кредита.     | `{"uid": "str", "features": {...}}` (пример)                                 | Заглушка: `{"message": "Prediction successfully"}`                | `415` - ошибка чтения `.pkl`<br>`500` - внутренняя ошибка               | Будет загружать модель из storage (`GET /get/{uid}`), десериализовать и предсказывать. |
| `POST`   | `/predict-from-csv`      | Принимает CSV‑файл (тестовую выборку) > возвращает `roc_auc` и предсказания. | `multipart/form-data`<br>`csv: UploadFile` (`.csv`)<br> + возможно `uid` в query | Заглушка: `{"message": "Prediction successfully"}`                | `415` - ошибка кодировки<br>`500` - ошибка чтения CSV                   | Получит модель по `uid`, применит ко всем строкам, вычислит метрики.               |
